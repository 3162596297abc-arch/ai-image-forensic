## 二、自定义事件在哪看

埋点事件：`upload_click`、`upload_success`、`analysis_start`、`analysis_finish`、`share_result`

- **实时验证**：报告 → 实时 → `按事件名称列出的事件数` 卡片，几秒内可见。
- **历史统计**：报告 → 生命周期 → 互动度 → **事件(Events)**（约 24 小时后聚合出现）。
- **转化漏斗**（上传点击 → 上传成功 → 分析开始 → 分析完成）：探索 → 选「漏斗探索」模板，依次拖入 4 个事件。

### ⚠️ 事件参数要变成可筛选维度，需先注册
事件自带的参数（`source`、`tier`、`method`、`file_type`、`file_size_kb`、`ai_participation`）默认**不能**在标准报告里当维度筛选。需手动注册一次：

> 管理(齿轮) → 数据显示 → **自定义定义** → 创建自定义维度 → 填入参数名（如 `source`）→ 保存。

注册后约 24–48 小时，即可在报告/探索里按 `source` 区分「hero / upload_device」等上传入口。

## 三、必做的一次性设置

1. **确认增强型衡量开启**（SPA 路由切换的 page_view 依赖它）：
   管理 → 数据流 → 选中 Web 数据流 → **增强型衡量** → 确保 `基于浏览器历史记录事件的页面更改` 勾选。✅ 默认开启，确认即可。
2. **关联 Google Search Console**（可选，看自然搜索关键词）：管理 → 产品关联 → Search Console。
3. **数据保留期拉满**：管理 → 数据显示 → 数据保留 → 改为 14 个月。

## 四、数据延迟须知
- **实时报告 / DebugView**：秒级，用于验证埋点。
- **标准报告（天/来源/地区/设备）**：通常延迟 **24–48 小时**才完整。上线第二天再看趋势才准。
